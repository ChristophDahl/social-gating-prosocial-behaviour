# Prosocial Behaviour as Socially Gated Action Selection

MATLAB code for the simulations, summaries, and figures accompanying the manuscript:

**Prosocial Behaviour as Socially Gated Action Selection: A Computational Reframing of Primate Prosociality**

The code implements a generative social-gating model in which prosocial behaviour is formalised as action selection under task, relational, and cost constraints. Outcome labels such as instrumental helping, low-cost prosocial choice, costly recipient benefit, strict altruism-like outcome, empathy-like responding, and reliable-partner collaboration are assigned after the simulated action outcome is known.

## MATLAB requirements

The code is written in MATLAB and uses only standard table, graphics, and file I/O functionality. No external toolboxes are required by the core simulation.

The scripts were written to avoid hard-coded local paths. The project root is inferred from the location of the MATLAB files.

## One-command reproduction

From the repository root, run:

```matlab
run('scripts/run_all.m')
```

This runs the simulation, writes CSV outputs to `results/`, and renders the main and supplementary figures.

## Running components separately

Run the full simulation and default diagnostic plots:

```matlab
run('matlab/main_social_gating_simulation.m')
```

Render only the two main manuscript figures from existing CSV outputs:

```matlab
addpath(genpath('matlab'))
render_main_figures_only(pwd)
```

Render the supplementary failure-decomposition figure:

```matlab
addpath(genpath('matlab'))
make_FigS1_failure_decomposition_by_system(pwd)
```

## Notes on terminology

The manuscript uses **prosocial behaviour** as the broad literature-level domain. Specific simulated profiles include instrumental helping, low-cost prosocial choice, costly recipient benefit, strict altruism-like outcomes, empathy-like responding, and reliable-partner collaboration. Some MATLAB variable names retain older development labels such as `helpingRate` or `costlyOtherBenefitRate` to preserve compatibility with existing output tables.
